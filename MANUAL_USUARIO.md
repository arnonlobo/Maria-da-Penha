# Manual do Usuário

## 1. Finalidade do sistema

O sistema foi desenvolvido para servir como ferramenta de apoio operacional na atuação policial em ocorrências de violência doméstica e familiar, com foco em consulta rápida, organização de dados e subsídio para a confecção do REDS.

O conteúdo da aplicação tem caráter auxiliar e não substitui normas institucionais, procedimentos legais, regulamentares ou a análise técnico-profissional da guarnição.

## 2. Acesso ao sistema

Ao abrir a aplicação, o utilizador visualiza a capa inicial, com acesso rápido às principais opções:

- `Entrar no Guia`
- `Baixar instrução em PDF`
- `Ver fluxograma`

## 3. Tela inicial

Após entrar no guia, a tela principal apresenta os acessos às funções do sistema:

- `1ª Resposta`
- `2ª Resposta`
- `Tipificações (Crimes)`
- `Modelos de REDS`
- `Perguntas Frequentes`

Também fica disponível, no topo, o botão de retorno à tela principal.

## 4. Fluxograma de atendimento

A opção `Ver fluxograma` permite abrir rapidamente a imagem do fluxo de atendimento para consulta no celular.

Uso recomendado:

- consultar a sequência de atuação antes ou durante o atendimento;
- tocar na imagem para abrir em tamanho maior;
- usar como apoio visual rápido.

## 5. Primeira resposta

O módulo `1ª Resposta` foi pensado para organizar os principais dados necessários ao atendimento inicial e ao subsídio da confecção do REDS.

O fluxo está dividido em 5 passos:

### 5.1 Controle da cena

Permite marcar pontos essenciais do primeiro atendimento:

- separação das partes;
- necessidade de socorro médico;
- flagrante delito ou quebra de medida protetiva;
- presença de crianças no local.

### 5.2 Qualificação

Permite preencher:

- dados da vítima;
- dados do autor;
- dados de testemunhas.

As testemunhas podem ser adicionadas e removidas conforme a necessidade.

### 5.3 Histórico

Reúne os campos para subsidiar a narrativa do REDS, sem repetir o conteúdo próprio do FONAR.

Os principais grupos de informação são:

- relação e contexto;
- perfil do autor;
- dinâmica e desfecho;
- atendimento e contexto do fato;
- lesões, ameaças, danos e provas;
- destino da vítima, situação do autor e encaminhamentos.

### 5.4 FONAR

Apresenta as 19 perguntas da avaliação de risco.

Cada pergunta deve ser respondida com:

- `Sim`
- `Não`
- `NS / NA`

O sistema só libera o passo seguinte após o preenchimento de todas as perguntas.

### 5.5 Relatório final

Ao final da `1ª Resposta`, o sistema apresenta:

- `Relatório estruturado` para cópia;
- `REDS padrão de referência`;
- `Prompt sugerido para IA`.

#### Relatório estruturado

Contém:

- qualificação das partes;
- testemunhas;
- subsídios para confecção;
- resultado do FONAR.

Este bloco foi mantido em formato copiável para facilitar o aproveitamento dos dados.

#### REDS padrão de referência

Serve como modelo textual para orientar a redação do histórico policial.

#### Prompt sugerido para IA

Permite copiar um comando pronto para utilização em sistemas de IA, com base nos dados preenchidos, para geração de uma minuta de histórico.

Importante:

- o texto produzido por IA deve sempre ser revisado pela guarnição;
- não se deve aceitar texto automatizado sem conferência técnica.

## 6. Segunda resposta

O módulo `2ª Resposta` foi pensado para apoiar a visita tranquilizadora e o acompanhamento pós-ocorrência.

O fluxo está dividido em 3 passos:

### 6.1 Qualificação

Permite preencher:

- número do REDS de origem;
- qualificação atualizada da vítima.

Essa etapa foi mantida porque pode haver alteração de contacto, endereço ou outros dados.

### 6.2 Visita tranquilizadora

Reúne os principais campos para o acompanhamento da vítima, como:

- data e hora da visita;
- local da visita;
- se a vítima foi localizada;
- forma de contacto;
- estado atual da vítima;
- existência de novo facto;
- MPU vigente;
- eventual descumprimento de MPU;
- local seguro;
- necessidade de apoio;
- risco elevado;
- relação íntima;
- resumo da visita;
- encaminhamento final.

### 6.3 Relatório final

Ao final da `2ª Resposta`, o sistema apresenta:

- `Relatório estruturado`;
- `Modelo padrão da 2ª resposta`;
- `Prompt sugerido para IA`.

O objetivo é permitir:

- copiar os dados organizados;
- consultar o modelo padrão da visita tranquilizadora;
- usar um prompt pronto para transformar os dados em minuta de histórico.

## 7. Tipificações (Crimes)

Esta área funciona como guia rápido de enquadramento jurídico.

Cada item apresenta:

- título do crime;
- categoria;
- pena;
- definição legal simplificada;
- verbos-chave;
- exemplos;
- confusões comuns;
- pergunta-chave.

Uso recomendado:

- consultar antes de fechar o histórico;
- usar como apoio de raciocínio;
- não substituir a análise jurídica do caso concreto.

## 8. Modelos de REDS

Esta área reúne modelos oficiais e práticos para consulta e cópia.

Os modelos disponíveis servem como base de redação para:

- `1ª Resposta - Padrão`
- `1ª Resposta - Com MPU`
- `2ª Resposta - Visita Tranquilizadora`

Cada modelo pode ser copiado diretamente.

## 9. Perguntas Frequentes

A secção `FAQ` reúne respostas rápidas sobre:

- primeira resposta;
- segunda resposta;
- terceira resposta;
- FONAR;
- REDS;
- medida protetiva;
- postura da guarnição;
- atuação em violência doméstica.

## 10. Botões e funções auxiliares

Ao longo do sistema, o utilizador encontra funções como:

- `Copiar Texto`
- `Copiar Modelo`
- `Copiar Prompt`
- `Imprimir`
- `Finalizar`

Além disso, há notificações visuais de confirmação após ações de cópia.

## 11. Recomendações de uso

- Preencher os dados de forma objetiva e técnica.
- Evitar duplicação desnecessária de informação.
- Rever o conteúdo antes de copiar para sistemas externos.
- Usar os modelos apenas como referência.
- Confirmar sempre os dados no contexto real da ocorrência.

## 12. Limitações

O sistema:

- não substitui os sistemas oficiais;
- não substitui a legislação nem as instruções institucionais;
- não valida juridicamente o conteúdo inserido;
- não dispensa a revisão humana.

## 13. Observação final

Esta aplicação deve ser utilizada como instrumento de consulta e apoio operacional, auxiliando a organização das informações e a redação inicial, sempre sob validação da guarnição responsável.
